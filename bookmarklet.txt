# 収集スクリプトの実行方法

note.comのCSP（Content Security Policy）により、ブックマークレットからの外部スクリプト
「読み込み」はブロックされます。ただしコードを全文埋め込んだブックマークレット
（bookmarklet-full.txt）の実行は可能です。用途に応じて以下から選んでください。

---

## PC: DevToolsコンソールに貼り付け（従来通り・推奨）

1. noteにログインし https://note.com/dashboard/stats を開く
2. note-analyzerのページで「収集スクリプトをコピー」ボタンを押す
3. F12でDevToolsを開き、Consoleタブにペースト → Enter
4. 収集完了後にJSONをコピーして、note-analyzerに貼り付けるか「Gistに保存」

※ ダッシュボード（/dashboard）で実行するとビュー数も取得できます。
　 プロフィールページで実行するとビューなしで収集されます。

---

## スマホ(Android): フルコード・ブックマークレット

1. note-analyzer の「📱 ブックマークレットとしてコピー（Android用）」を押す
   （アプリがその場で javascript: URL を生成してクリップボードに入れる。
   　bookmarklet-full.txt の中身をコピーしても同じ）
2. Chromeで適当なページをブックマーク → ブックマーク編集でURLを↑に置き換え、
   名前を「note収集」などにする
3. noteにログインした状態で https://note.com/dashboard/stats を開く
4. アドレスバーに「note収集」と入力し、候補に出たブックマークをタップ
   （※アドレスバー経由で呼ぶのがコツ。ブックマーク一覧から開くと動かない場合がある）
5. 画面に進捗オーバーレイが出る。完了したら「コピー」でJSONを取得し、
   note-analyzerに貼り付け

## スマホ(iPhone): ショートカット＝モバイル版コンソール

iOSの「ショートカット」アプリの「Webページで JavaScript を実行」アクションは、
共有シートから呼び出すと「いま表示しているSafariのページ上」でJSを実行します。
実行コンテキストはそのページ本体（同じDOM・同じオリジン・同じログインCookie。
HttpOnlyのセッションCookieもfetchに自動で付く）なので、開発者コンソール貼り付けと
完全に等価です。

### 初回セットアップ（1回だけ）

1. iPhoneのSafariで note-analyzer を開き、「収集スクリプトをコピー」を押す
   （collector.js全文＋末尾に completion() 呼び出しを同梱した共用版が
   クリップボードに入る。手での追記は不要）
2. ショートカットアプリ → 新規ショートカット → アクション検索で
   「JavaScript」→「Webページで JavaScript を実行」を追加
3. スクリプト欄をタップし、入っているサンプルコードを全部消して、
   1でコピーしたものをそのまま貼り付け
   ※ completion() はショートカットのタイムアウト回避用（同梱済み）。
   　 collector.js は起動後すぐ制御を返し、収集は非同期でページ内で続くので、
   　 ショートカットは一瞬で「完了」し、収集はページで走り続ける。
   　 PCコンソールに同じものを貼っても動く（completion未定義はtryで無害化）
4. 設定（下部のiボタン/共有シートアイコン）→「共有シートに表示」をON。
   受け取る内容は「Safari Webページ」だけにしておくと共有シートが散らからない
5. 名前を「note収集」などにして保存

### 毎回の実行

1. Safariでnoteにログインし https://note.com/dashboard/stats を開く
   （statsで実行するとビュー数つき。プロフィールページだとビューなし）
2. 共有ボタン →「note収集」をタップ
3. 初回のみ「"note.com"でJavaScriptの実行を許可しますか？」→ 許可
4. ページ上に進捗オーバーレイが出る。収集が終わるまでSafariを前面のまま、
   画面を消灯させないこと（バックグラウンドではJSが止まる。記事数×
   スキ一覧の取得で数分かかる）
5. 完了画面の「コピー」でJSONを取得 → note-analyzerのタブに切り替えて
   貼り付け、または「Gistに保存」

### collector.js を更新したら

ショートカット内のスクリプトは保存時のコピーなので自動では追従しません。
更新時はスクリプト欄を貼り直してください（手順1と3だけやり直し）。

※ Safariのブックマークレット（Android手順と同じURL置き換え）でも動きます。
　 お気に入り編集でURL欄に28KB貼るのがつらい場合はショートカット推奨。

## GitHub Actions について（非推奨・参考）

「Actionsで定期収集してGistに自動保存」は技術的には組めますが、以下の理由で
スマホ運用の主手段には向きません。

- note.comのAPIはログインセッションCookie（HttpOnly）が必要。HttpOnlyなので
  ブックマークレットからも読めず、結局PCのDevToolsで一度Cookieを抜いて
  Secretsに登録する必要がある（＝スマホ完結にならない）
- セッションが切れるたびに再登録が必要で、切れたことに気づきにくい
- アカウントのCookieをCIに預けるリスクと、データセンターIPからのAPI連打が
  bot検知に引っかかるリスク

上のブックマークレット／ショートカットはスマホ内で完結し、Cookieも端末外に
出ないため、こちらを推奨します。

---

## メンテナンス

collector.js を更新したら `node make_bookmarklet.js` で bookmarklet-full.txt を
再生成してください（コメント除去＋URLエンコードのみの保守的な変換で、
変換後コードは構文検証されます）。
